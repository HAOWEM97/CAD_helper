# CAD 自动布线与通道配置 Web

基于 `React + TypeScript + Vite` 的 Local-First CAD 自动布线与通道配置工具。

当前版本已完成阶段 5 初版：CAD 脚本导出，并已通过本地验证。

最近完成事项：

- 基础项目配置
- Redux 状态结构
- 非线性步骤导航
- 工作台四区布局
- PNG 底图导入与大图查看
- 左键标记参考点，右键或中键拖拽平移，滚轮缩放
- 两点参考点校准，参考点保存为底图像素坐标
- X/Y 独立像素到 CAD 转换比
- 状态栏 CAD 坐标显示
- 高倍率缩放黑屏修复，图纸放大后仍保持可见
- 本地草稿自动保存与页面刷新恢复
- 拓扑节点与通道连续绘制
- 正交锁定、节点吸附、轴线吸附
- 绘制 / 选择模式切换
- 节点拖动、节点与通道删除
- 通道类型编辑：线槽、排管
- 设备接线孔设置：设备实例或自定义接线孔、接线孔类型、线缆型号、数量和安装高度
- 常用库：线缆规格、接线孔模板、设备类型模板，工程内保存与浏览器全局复用
- 库管理页：线缆可新增、编辑、删除，接线孔模板可编辑并按确认同步到当前工程节点
- 路由代办：有限线缆接线孔作为起点，不限接线孔作为承接终点
- 路由页支持行内选择终点、删除路由和按原起终点重算需重算路由；同一起点重复生成会替换旧路由
- 已生成路由可点击展开，显示该路由的二维平面长度，便于配合图纸核查路径
- 右侧属性面板支持拖拽调宽，宽度随本地草稿恢复
- 基于拓扑网络的二维最短路径路由和线缆兼容性校验
- 生成路由后高亮经过通道，并将线缆编号写回通道
- 设备页支持一键清除所有节点属性设定；清除接线孔、设备实例、路由和通道线缆登记，但保留拓扑与常用库
- 根据有效路由内线缆自动推演线槽/排管规格
- 规格生成后解锁通道敷设深度输入
- 左侧 BOM 实时统计线缆工程用量（*1.05）、通道规格数量和通道长度
- 算量页支持导出线缆用量明细 CSV，按路由和线缆型号/用途拆分工程长度
- 导出页和顶部导出按钮支持生成 `.bas` CAD VBA 脚本
- 脚本按 CAD 坐标输出通道，线槽、排管和标注分别写入独立图层
- 排管导出包含规格文字标注，脚本注释保留通道 id 和规格，便于复查
- 已保留 `.scr` 脚本生成接口，后续可扩展为正式交付入口

## 当前实施细节

- 设备不再直接等同于拓扑节点；节点保存的是 `DeviceConnectionPoint`，即某台设备的某个接线孔。
- 同一台设备可绑定多个接线孔节点，例如 `主机1 / 主机到储能` 和 `主机1 / 主机到终端`。
- `常见设备接线.xlsx` 已固化为默认库，包含设备类型、所属区段、线缆型号、用途、数量、外径和每种线缆的安装高度。
- 常用库分为线缆规格、接线孔模板、设备类型模板三层，并同时支持工程内保存和浏览器全局复用。
- 路由页按接线孔生成待办；有限数量接线孔作为起点，不限数量接线孔只作为承接终点。
- 线缆库只保存型号和外径；用途属于接线孔明细中的可选字段。
- 线缆兼容性按“型号 + 数量”校验；用途只用于展示和人工核对。
- 默认设备库包含“电缆井 / 电缆汇总点”，可作为任意有限线缆的不限承接终点。
- 当前业务规则是一个起点接线孔只保留一条路由；重新选择终点或重算会替换旧路由，避免重复登记。
- BOM 不落库，按当前有效路由、拓扑长度、安装高度和通道深度派生；通道深度是用户输入并随工程保存。
- 页面 BOM 中的线缆汇总显示工程用量，口径为“路由平面长度 + 起终点接线长度 + 路径高度变换长度”后乘以 `1.05` 工程放量系数，再乘以线缆数量。
- 线缆用量明细 CSV 按“有效路由 + 起点线缆明细”逐行导出，字段和详细计算口径见 [docs/channel-spec-rules.md](./docs/channel-spec-rules.md)。
- CAD 脚本导出不落库，按当前校准、拓扑、通道类型和规格实时生成；未完成校准或无有效拓扑时禁止导出。
- `.bas` 输出包含 `CAD_HELPER_TRAY`、`CAD_HELPER_DUCT` 和 `CAD_HELPER_ANNOTATION` 三个图层；线槽输出为轻量多段线，排管输出为中心线并在中点添加规格文字。
