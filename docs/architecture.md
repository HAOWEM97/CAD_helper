# Architecture

本项目采用 Local-First 前端架构。浏览器负责图纸读取、交互绘制、拓扑状态、路由计算、BOM 派生计算、工程保存与 CAD 脚本生成。

## 核心原则

- Redux 状态只保存可序列化业务数据。
- Canvas、OpenSeadragon、DOM 引用等运行时对象必须留在组件或服务实例中。
- BOM、路由状态、导出脚本应由当前工程状态派生或生成，避免保存不可追踪的重复数据。
- 工作流采用非线性模式，用户可随时切换阶段并返回修改。
- 设备建模采用“设备实例 + 接线孔节点”两层结构，避免把一个设备的多个出线孔误建成多台设备。
- 常用库分为工程内库与浏览器全局库；工程内库随项目状态保存，全局库用于跨工程复用。

## 页面结构

- 顶部工具栏：项目身份、步骤导航、保存和导出入口。
- 左侧面板：BOM、图层控制、对象列表。
- 中央画布：底图、校准、拓扑、设备、路由和轮廓的主交互区域。
- 右侧面板：当前选中对象属性、设备接线孔编辑、路由代办和阶段规则；展开宽度可拖拽调整并保存在本地草稿 UI 状态。
- 底部状态栏：CAD 坐标、缩放、正交、吸附和校准状态。

## 阶段 3 页面行为

- 设备页点击拓扑节点后设置设备实例或自定义接线孔，并配置线缆型号、数量和接线点高度。
- 已设置为设备接线孔的节点在画布中使用独立颜色显示。
- 设备页核对表显示所有接线孔，并提供一键清除所有节点属性设定。
- 路由页按状态展示接线孔：待路由、已完成起点、已作为终点、不限承接端。
- 不限数量接线孔不作为主动起点，只作为可承接终点。
- 同一起点接线孔重复生成路由时替换旧路由，防止同一起点在“已生成路由”中出现多条当前记录。
