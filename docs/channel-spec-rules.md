# 通道规格推荐规则

本文档记录当前阶段 4 已应用的线槽、排管规格推荐、用户确认、容积率校核和线缆分配规则。

## 规格确认

- 系统先根据有效路由中的线缆负载推荐通道规格。
- 推荐规格不是最终规格；用户点击“确认推荐”、选择常见规格或保存自定义规格后，才成为该通道的最终规格。
- 线缆负载变化后，系统保留用户已选最终规格，但标记为“需复核”，并重新计算容积率与警告。
- BOM 的通道规格统计优先使用最终规格；未确认时使用当前推荐规格。

## 默认高度

- 通道高度使用 `depthMm` 字段保存，单位为 `mm`；负数表示通道布置在地下。
- 用户选择规格或保存自定义规格时，如果该通道高度为空，系统会自动写入默认高度。
- 用户已经手动修改过高度时，切换规格不覆盖用户输入。
- 线缆长度和工程用量计算继续使用用户最终保存的通道高度。

默认高度规则：

- `16*DN125`：`-1190mm`
- `12*DN125`：`-1035mm`
- `8*DN125`：`-880mm`
- `2*DN125+2*DN32`：`-500mm`
- 所有常见线槽：`0mm`
- 自定义排管：`-500mm`
- 自定义线槽：`0mm`

## BOM 与线缆用量计算口径

阶段 4 的 BOM 和线缆明细均由当前工程状态实时派生，不写入工程数据。修改路由、设备接线高度、通道高度或线缆数量后，页面显示和导出结果会重新计算。

### 路由基础长度

- `路由长度` 使用有效路由经过通道的 CAD 平面二维长度之和，单位内部为 `mm`。
- 通道平面长度由通道两端拓扑节点的 CAD 坐标计算，不使用屏幕像素坐标。
- 路由状态为 `needs-recalculation` 时，不参与 BOM 汇总和线缆明细 CSV 导出。
- 起点接线孔中数量为“不限”的线缆不作为主动用量统计行；不限接线孔只作为承接终点。

### 竖向长度拆分

每条有效路由中的每一种有限数量线缆，按以下方式拆分竖向长度：

- `起点接线长度 = |起点接线高度 - 第一段通道高度|`
- `终点接线长度 = |终点接线高度 - 最后一段通道高度|`
- `路径高度变换长度 = 路径中相邻通道高度差绝对值之和`
- 如果终点是不限承接点或未找到完全匹配的终点线缆明细，终点接线高度使用起点线缆的接线高度暂算。

### 单根长度与工程用量

- `单根长度 = 路由长度 + 起点接线长度 + 终点接线长度 + 路径高度变换长度`
- `工程放量系数 = 1.05`
- `单根工程长度 = 单根长度 * 1.05`
- `总计工程长度 = 单根工程长度 * 线缆数量`
- 页面左侧 `BOM 清单` 和右侧 `BOM 摘要` 中的线缆汇总均显示工程用量，即已经乘以 `1.05` 后的结果。
- 通道 BOM 仍统计通道自身平面长度和规格数量，不乘以线缆工程放量系数。
- 页面显示单位为 `m`，内部计算仍使用 `mm`；页面汇总保留 2 位小数，CSV 明细保留 3 位小数。

### 通道高度缺失

- 页面 BOM 在通道高度缺失时仍可暂算，缺失通道按 `0mm` 高度参与竖向长度计算，并显示提示。
- 线缆用量明细 CSV 用于交付复核，要求有效路由经过的通道均已填写高度；若存在缺失高度，导出按钮不可用并提示补齐。

### 线缆用量明细 CSV

CSV 每行表示“一条有效路由中的一种起点线缆明细”，字段顺序为：

- `路由起点`
- `路由终点`
- `路由长度(m)`
- `线缆型号`
- `线缆用途`
- `起点接线长度(m)`
- `终点接线长度(m)`
- `路径高度变换长度(m)`
- `单根长度(m)`
- `工程放量系数`
- `单根工程长度(m)`
- `线缆数量`
- `总计工程长度(m)`

CSV 使用 UTF-8 BOM，便于 Excel 直接打开中文表头。

## 线缆分类

- 接线孔明细中的用途包含“通信线”时，该线缆归为通信线。
- 其他用途都归为配电线，包括交流线、压缩机线、直流线、接地线、电源线等。
- 分类用于容积率校核和排管分配；具体用途仍会在明细中展示。

## 线槽规则

常见线槽规格：

- `200x150mm`
- `300x150mm`
- `500x150mm`

通信线和配电线同时存在时，推荐带分隔板线槽：

- `200x150mm（带分隔板150/50）`
- `300x150mm（带分隔板250/50）`
- `500x150mm（带分隔板450/50）`

容积率规则：

- 通信仓容积率低于 `50%`。
- 配电仓或普通线槽容积率低于 `40%`。
- 同时包含通信线和配电线但未使用分隔板时，显示警告。

## 排管规格

常见管型：

- `DN125 CPVC`：默认壁厚 `6mm`，内径 `113mm`。
- `DN100 CPVC`：默认壁厚 `5mm`，内径 `90mm`。
- `DN32 钢管`：内径 `37.4mm`。

常见组合：

- `16*DN125`
- `12*DN125`
- `8*DN125`
- `2*DN125+2*DN32`

## 排管分配

- 通信线必须走 `DN32`。
- 配电线可以走 `DN125`、`DN100`，也可以走空闲 `DN32`。
- `DN32` 被通信线占用后，不再分配给配电线。
- 相同线缆型号可以走同一根管道。
- 直流线和接地线可以走同一根管道。
- 交流线不能仅因为同属配电线就和直流线、接地线混管；除非同时满足“相同线缆型号”规则。
- 分配时优先使用大管，接近容积率上限时放入下一根兼容管。
- 缺少可独立敷设的管道时，显示明确警告。

容积率规则：

- 通信线在 `DN32` 中容积率低于 `50%`。
- 配电线在任意管道中容积率低于 `40%`。

## 展示规则

- 每根管道显示该管内线缆型号、用途、数量和该管容积率。
- 通道展开后的线缆明细按“型号 + 用途 + 外径 + 分类”合并统计显示。
- 总截面按当前通道内所有有效路由线缆汇总。
