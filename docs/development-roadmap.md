# Development Roadmap

开发严格按 `Plan.md` 的阶段推进。

## 当前阶段

阶段 5：CAD 脚本导出。当前已完成本地初版实现，基于当前校准、拓扑、通道类型和规格生成 `.bas` CAD VBA 脚本，并保留 `.scr` 生成接口。

验收重点：

- 拓扑节点可以设置为设备接线孔，并保存设备实例或自定义接线孔、线缆型号、数量和安装高度。
- 自定义线缆规格、接线孔模板、设备类型模板可保存到当前工程和浏览器全局库。
- 线缆库只维护型号和外径，接线孔明细维护可选用途、数量和安装高度。
- 库页可管理工程库，并支持从全局导入或同步到全局。
- 同类型设备默认使用“类型 + 最小空缺编号”命名，用户仍可自定义名称。
- 路由页按接线孔生成待办，不限数量接线孔只作为承接终点。
- 路由页支持起点行内选择终点、删除路由和重算需重算路由；同一起点重复生成路由时替换旧记录。
- 已生成路由可点击展开，显示二维平面路由长度，供用户配合图纸核查。
- 右侧属性面板支持拖拽调宽，并通过本地草稿恢复用户上次宽度。
- 默认“电缆井 / 电缆汇总点”可作为任意有限线缆的不限承接终点。
- 路由按 CAD 平面二维长度计算最短路径，并高亮经过通道。
- 网络断开时不生成错误路由。
- 设备页提供一键清除所有节点属性设定；清除设备实例、接线孔、路由和通道线缆登记，但保留拓扑和常用库。
- 线槽按线缆截面积和填充率推演规格。
- 排管按线缆数量推演排布矩阵。
- 有有效路由线缆的通道解锁敷设深度输入。
- BOM 按二维路径长度、安装高度、通道深度和深度变化实时派生，线缆汇总显示工程用量（*1.05）。
- 算量页支持导出线缆用量明细 CSV，明细按有效路由和起点线缆明细拆分，并展示路由长度、接线长度、路径高度变换长度、工程放量系数和总计工程长度。
- 导出页支持校验、预览并下载 `.bas` 脚本。
- 顶部“导出 CAD 脚本”按钮可直接生成 `.bas` 文件，并在导出前切换到导出页。
- 线槽输出到 `CAD_HELPER_TRAY` 图层，使用轻量多段线表达闭合空心边界；边界宽度取用户确认的线槽规格宽度，拼接节点输出并集外轮廓。
- 排管输出到 `CAD_HELPER_DUCT` 图层，使用中心线表达，并在中点写入规格文字标注。
- 标注输出到 `CAD_HELPER_ANNOTATION` 图层，脚本注释保留通道 id、类型和规格。
- 未完成校准或无有效通道拓扑时，不生成脚本。

## 本轮对话确认的产品规则

- 节点表示设备接线孔，不表示整台设备。
- 用户所有自定义都可加入常用库：线缆规格、接线孔模板、设备类型模板。
- 常用库保存范围为当前工程和浏览器全局库。
- 自定义设备常用库按设备类型模板复用，套用后仍使用名称前缀自动编号。
- 一个接线孔的明细线缆在本阶段整组路由，不拆分到多个终点。
- 一个有限起点接线孔当前只允许保留一条路由；未来如需一处起点拆分到多个终点，应另开“拆分路由”能力。
- 已作为终点或已完成路由的接线孔不从页面消失，而是在路由页状态分组中保留，便于核对。
