# Development Roadmap

开发严格按 `Plan.md` 的阶段推进。

## 当前阶段

阶段 3：设备接线孔、常用库与路由代办。当前已完成本地实现并提交到 `main`，提交为 `05e9d81 feat: support device connection point routing`。

验收重点：

- 拓扑节点可以设置为设备接线孔，并保存设备实例或自定义接线孔、线缆型号、数量和接线点高度。
- 自定义线缆规格、接线孔模板、设备类型模板可保存到当前工程和浏览器全局库。
- 同类型设备默认使用“类型 + 最小空缺编号”命名，用户仍可自定义名称。
- 路由页按接线孔生成待办，不限数量接线孔只作为承接终点。
- 路由按 CAD 平面二维长度计算最短路径，并高亮经过通道。
- 网络断开时不生成错误路由。
- 设备页提供一键清除所有节点属性设定；清除设备实例、接线孔、路由和通道线缆登记，但保留拓扑和常用库。

## 本轮对话确认的产品规则

- 节点表示设备接线孔，不表示整台设备。
- 用户所有自定义都可加入常用库：线缆规格、接线孔模板、设备类型模板。
- 常用库保存范围为当前工程和浏览器全局库。
- 自定义设备常用库按设备类型模板复用，套用后仍使用名称前缀自动编号。
- 一个接线孔的明细线缆在本阶段整组路由，不拆分到多个终点。
- 已作为终点或已完成路由的接线孔不从页面消失，而是在路由页状态分组中保留，便于核对。
