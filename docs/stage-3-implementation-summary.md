## 本轮对话形成的关键决策

- 节点不再代表整台设备，而是代表某台设备的某个接线孔。
- 同一台设备可以有多个接线孔节点，例如 `主机1 / 主机到储能` 与 `主机1 / 主机到终端`。
- `常见设备接线.xlsx` 中的 `所属区段` 作为接线孔类型和默认接线孔模板名称。
- Excel 默认库已固化到代码中，作为初始设备类型模板、线缆规格和接线孔明细来源。
- 用户所有自定义均可加入常用库，包括线缆规格、接线孔模板、设备类型模板。
- 常用库同时保存到当前工程和浏览器全局库。
- 常用设备保存为设备类型模板，不保存为固定设备实例；套用后继续按名称前缀自动编号。
- 一个接线孔的明细线缆在阶段 3 整组路由，不拆分到多个终点。
- 路由页不隐藏已处理节点，而是以状态分组保留，便于核对遗漏。
- 数量为“不限”的接线孔只作为承接终点，不作为主动路由起点。

## 已实现内容

- 领域模型升级：
  - `DeviceInstance`：真实设备实例。
  - `DeviceConnectionPoint`：绑定拓扑节点的设备接线孔。
  - `CableSpec`：线缆规格，包含用途、型号、外径原文、外径范围和默认外径。
  - `ConnectionCableItem`：接线孔内某种线缆的数量和接线点高度。
  - `ConnectionPointPreset`：常用接线孔模板，包含一组接线孔线缆明细。
  - `DeviceTypePreset`：设备类型模板，包含接线孔类型和默认线缆明细。
- 默认库：
  - 从 `常见设备接线.xlsx` 固化 7 类设备类型。
  - 固化 11 类线缆型号。
  - 支持解析 `约 19.5 - 20.5` 这类外径范围，并取中位数作为默认外径。
- 设备页：
  - 点击拓扑节点后设置设备实例或自定义接线孔、接线孔类型、线缆型号、数量和接线点高度。
  - 支持选择已有设备实例，把多个节点归属到同一设备。
  - 支持添加/编辑自定义接线孔内的线缆、数量和接线点高度。
  - 支持把自定义线缆、接线孔模板、设备类型模板加入常用库。
  - 提供接线孔核对表。
  - 已设置为设备接线孔的节点在画布中变色显示。
  - 提供“一键清除所有节点属性设定”按钮。
- 路由页：
  - 按接线孔生成待办。
  - 按状态分组：待路由、已完成起点、已作为终点、不限承接端。
  - 选择起点后过滤/展示兼容终点。
  - 生成路由后高亮经过通道，并把线缆摘要写回通道。

## 清除按钮行为

设备页的“清除所有节点属性设定”会清空：

- `deviceInstances`
- `connectionPoints`
- `routes`
- 通道上的线缆登记摘要

不会清空：

- 拓扑节点和通道
- 默认线缆/设备库
- 用户保存的常用线缆规格、接线孔模板、设备类型模板

## 路由校验规则

- 有限数量到有限数量：
  - 用途/类型名、型号、数量必须一一匹配。
- 有限数量到不限数量：
  - 终点必须存在相同用途/型号的不限承接能力。
- 不限数量接线孔：
  - 不生成主动起点待办。
- 不兼容时：
  - 不允许生成路由，并显示不兼容原因。
