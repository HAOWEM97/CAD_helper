# Plan.md：CAD 自动布线与通道配置 Web 项目开发计划

## 1. 项目背景

本项目旨在开发一个基于 `React` 的纯前端网页版工具，用于辅助 CAD 自动布线和通道配置。用户通过导入 CAD 图纸导出的 `PNG` 底图，在网页中完成坐标校准、通道网络绘制、设备节点配置、线缆路由、材料统计以及 CAD 脚本生成。

项目采用 `Local-First` 架构，所有图纸读取、交互绘制、路径计算、材料清单统计、工程保存和 CAD 脚本生成均在浏览器本地完成，不依赖业务后端，避免图纸数据外泄。

## 2. 项目目标

- 支持用户导入超大尺寸 `PNG` 图纸，并进行流畅缩放、拖拽和局部查看。
- 支持通过两个基准点完成屏幕像素坐标到 CAD 真实坐标的映射。
- 支持用户在底图上绘制通道拓扑网络，并区分线槽、排管等通道类型。
- 支持将节点配置为设备节点，并录入设备接线高度。
- 支持基于拓扑网络生成线缆最短路径。
- 支持根据线缆路径、设备高度、通道深度动态统计线缆长度和材料清单。
- 支持生成可供 AutoCAD/ZWCAD 使用的 `.bas` 或 `.scr` 文本脚本。
- 支持工程 `.json` 本地保存和再次载入。
- 支持用户在任意阶段返回之前步骤修改，并触发相关数据重新计算或失效提示。

## 3. 执行约束

### 3.1 第一性原理

- 每个功能必须从真实业务目标出发，而不是为了技术完整性而增加复杂功能。
- 如果需求动机、业务目标或交互边界不清晰，应先暂停并确认，不应自行扩展。
- 不假设用户已经完全明确最终形态，开发过程中应持续校验需求和实现是否仍然服务于核心目标：辅助 CAD 布线、算量和脚本生成。

### 3.2 方案规范

- 不允许采用兼容性补丁式方案。
- 不允许为了覆盖所有可能性而过度设计。
- 不允许加入用户未提出且会改变业务逻辑方向的兜底、降级或扩展方案。
- 每个阶段必须完成端到端验证后再进入下一阶段。
- 所有核心数据必须可追踪、可验证、可重新计算。
- 前端状态不得保存不可序列化的渲染对象、DOM 对象或第三方实例。

## 4. 推荐技术栈

- 框架：`React + TypeScript + Vite`
- 状态管理：`Redux Toolkit`
- 大图查看：`OpenSeadragon`
- 绘制层：`Canvas` 或 `SVG Overlay`
- 几何计算：项目内封装独立 `geometry` 模块
- 路径算法：项目内实现 `Dijkstra` 或 `A*`
- 文件导入导出：`File API`、`Blob`、`URL.createObjectURL`
- 测试：单元测试、交互测试、端到端业务验收

## 6. 页面布局规划

工作台页面采用非线性工作流，不做只能前进不能返回的向导式流程。

```text
顶部：全局工具栏 / 步骤导航 / 保存 / 导出
左侧：BOM 清单 / 图层控制 / 对象列表
中间：图纸交互画布
右侧：属性编辑面板
底部：状态栏 / 鼠标 CAD 坐标 / 当前模式 / 吸附状态
```

各区域职责：

- 中间图纸区：底图查看、坐标校准、通道绘制、设备放置、路由查看。
- 左侧 BOM/图层区：材料清单统计、图层显示隐藏、对象快速定位。
- 右侧属性面板：编辑当前选中对象的类型、高度、深度、规格、线缆信息。
- 底部状态栏：显示 CAD 坐标、缩放比例、正交锁定、吸附状态、当前步骤。

用户必须可以返回之前步骤修改：

- 修改校准后，应重新换算坐标并提示检查已有拓扑。
- 修改拓扑后，受影响路由应标记为需要重新计算。
- 修改设备高度或通道深度后，BOM 应立即重算。
- 修改通道类型后，规格推演和 CAD 导出样式应同步变化。
- 删除已参与路由的节点或通道时，应明确提示其影响范围。

## 7. 分阶段开发计划与验收项

### 阶段 0：项目骨架与基础工作台

开发内容：

- 初始化 `React + TypeScript + Vite` 项目。
- 建立基础目录结构。
- 配置状态管理、路由、全局样式和基础布局。
- 实现工作台四区布局：左侧 BOM/图层区、中间图纸区、右侧属性面板、底部状态栏。
- 建立非线性步骤导航：校准、绘制、设备、路由、算量、导出。

测试与验收：

- 页面能够正常启动和加载。
- 四个主区域在桌面分辨率下布局稳定。
- 步骤导航可以自由切换，不限制用户只能向后执行。
- 空项目状态下，各面板显示清晰的空状态。
- Redux 状态树中不存在 DOM、Canvas、OpenSeadragon 实例等不可序列化对象。

### 阶段 1：底图导入与坐标校准

开发内容：

- 支持用户导入本地 `PNG` 底图。
- 使用大图查看组件实现缩放和拖拽，图纸区交互采用固定分工：左键用于标记、选择、修改参考点等业务操作；右键按住拖拽和中键按住拖拽用于平移视图；滚轮滚动用于以鼠标位置为中心缩放。
- 图纸区应禁用默认浏览器右键菜单和中键自动滚屏，避免干扰 CAD 式图纸操作。
- 视图导航在阶段 1 的所有状态下始终可用。用户标记第一个参考点后，可以拖拽或缩放图纸再标记第二个参考点；标记完成后，也可以继续拖拽、缩放或重新修改任意参考点落点。
- 支持点击两个基准点并输入对应 CAD 绝对坐标。
- CAD 绝对坐标输入必须支持小数、负数和很小的数值，不得因浏览器数字输入控件限制阻止用户输入有效 CAD 坐标。
- 参考点必须保存为底图像素坐标，不保存屏幕坐标或当前缩放、平移后的临时坐标。
- 建立 `pixel -> CAD` 和 `CAD -> pixel` 坐标转换模块，对外提供纯函数能力，内部按两点向量计算缩放、旋转和平移，并显式处理图片像素 Y 轴向下与 CAD 坐标习惯之间的关系。
- 当前两点校准默认采用统一比例换算；如果实际图纸存在横向和纵向比例不一致，应规划为三点校准或 X/Y 两条基准线校准，分别确定 `scaleX`、`scaleY` 和坐标方向，不应由任意两点强行推断非等比例变换。
- 图纸区左下角应提供小型导航框，显示整张底图缩略图和当前视野位置，帮助用户在放大查看局部时确认所在区域。
- 底部状态栏实时显示鼠标所在 CAD 坐标。

测试与验收：

- 用户可成功导入 `PNG` 图片。
- 大尺寸图纸缩放、拖拽流畅。
- 右键按住拖拽和中键按住拖拽均可平移图纸，滚轮可缩放图纸，左键不会触发视图平移。
- 标记第一个参考点后，用户可以拖拽或缩放图纸，再标记第二个参考点。
- 两个参考点标记完成后，用户仍可拖拽或缩放图纸，并可重新修改任意参考点落点。
- CAD 坐标输入支持 `3003.25`、`-180.75`、`0.001` 等小数值，并能参与校准计算。
- 两点校准后，基准点换算结果与输入 CAD 坐标一致。
- 放大或平移图纸时，左下角导航框同步显示当前被查看的图纸区域。
- 无论当前视图缩放和平移如何变化，参考点位置、CAD 坐标换算结果和状态栏坐标显示保持一致。
- 鼠标移动时，状态栏 CAD 坐标实时变化。
- 重新校准后，系统能提示用户已有拓扑和路由可能受影响。

### 阶段 2：拓扑网络绘制

开发内容：

- 在图纸区绘制节点和通道连线。
- 支持正交锁定、节点吸附、轴线吸附。
- 支持选中、删除、移动节点和通道。
- 支持设置通道类型：线槽、排管。
- 未完成算量前，通道规格和敷设深度字段保持锁定。

测试与验收：

- 用户可以连续点击绘制通道网络。
- 正交锁定开启后，绘制线段只能沿水平或垂直方向生成。
- 鼠标靠近已有节点时可自动吸附。
- 选中通道后，右侧属性面板显示通道类型。
- 未进入规格推演前，规格和深度输入不可编辑。
- 删除节点或通道时，关联拓扑结构保持一致。

### 阶段 3：设备节点与路由规划

开发内容：

- 支持将拓扑节点标记为设备。
- 支持录入设备名称、类型和接线点高度。
- 支持创建单根线缆和线缆组合模板。
- 实现基于拓扑图的最短路径计算。
- 将路由结果写入经过的通道。
- 在图纸区高亮显示生成路径。

测试与验收：

- 节点可被成功标记为设备。
- 设备高度修改后状态正常保存。
- 用户选择起点、终点和线缆模板后可以生成路由。
- 简单网络中，算法返回预期最短路径。
- 网络断开时，系统不生成错误路由，并明确提示无法到达。
- 修改已参与路由的通道后，相关路由被标记为需要重新计算。

### 阶段 4：规格推演与动态 BOM

开发内容：

- 根据通道内线缆统计推荐通道规格。
- 线槽根据线缆截面积和填充率推算推荐尺寸。
- 排管根据线缆数量推算排布矩阵。
- 规格生成后解锁敷设深度输入。
- BOM 根据二维路径长度、设备高度、通道深度动态计算总长度。
- 左侧 BOM 面板实时刷新材料统计。

测试与验收：

- 通道内线缆变化后，推荐规格同步变化。
- 规格生成前深度不可编辑，规格生成后深度可编辑。
- 修改设备高度后，相关线缆总长度立即重算。
- 修改通道深度后，相关线缆总长度立即重算。
- BOM 中线缆长度、通道数量、规格统计与当前图纸状态一致。
- 多次修改后，不出现旧数据残留或重复统计。

### 阶段 5：CAD 脚本导出

开发内容：

- 将线槽转换为 CAD 多段线或边界脚本。
- 将排管转换为 CAD 中心线、虚线图层和文字标注。
- 将 CAD 坐标用于脚本输出，不输出屏幕像素坐标。
- 支持导出 `.bas`，保留 `.scr` 结构接口。
- 使用浏览器本地下载，不请求服务器。

测试与验收：

- 点击导出后可以生成文本脚本文件。
- 脚本中坐标值来自校准后的 CAD 坐标。
- 线槽和排管输出到不同图层或使用不同线型表达。
- 排管脚本包含规格文字标注。
- 未完成校准或无有效拓扑时，不允许导出无意义脚本。
- 导出的脚本文本结构可读、可追踪、可复查。

### 阶段 6：工程保存与恢复

开发内容：

- 支持将工程状态导出为 `.json`。
- 工程 JSON 仅保存业务数据和数学数据。
- 支持从 `.json` 载入工程。
- 载入后恢复拓扑、设备、路由、规格和 BOM。
- 对底图文件引用进行明确处理，避免误以为浏览器永久保存原图。

测试与验收：

- 用户可以导出工程 JSON。
- 导出的 JSON 不包含 DOM、Canvas、OpenSeadragon 等运行时对象。
- 重新导入 JSON 后，拓扑、设备、通道、路由与导出前一致。
- 重新导入后 BOM 重新计算结果与导出前一致。
- 若底图需要用户重新选择，系统应明确提示。
- 多次保存和恢复后，数据不重复、不丢失、不错位。

## 8. 核心数据模型草案

```ts
type CadPoint = {

  y: number;

type ImageMetadata = {
  id: string;

type CalibrationState = {
  imagePointA: CadPoint;
  imagePointB: CadPoint;
  cadPointA: CadPoint;
  cadPointB: CadPoint;
  scale: number;
  rotationRadians: number;
  transform: {
    originImagePoint: CadPoint;
    originCadPoint: CadPoint;
    imageYAxis: 'down';
    a: number;
    b: number;
  };

type Project = {
  id: string;
  name: string;
  image: ImageMetadata | null;
  calibration: CalibrationState | null;
  topology: TopologyGraph;
  devices: DeviceNode[];

  id: string;
  startNodeId: string;
  endNodeId: string;
  category: 'tray' | 'duct';

};

type CableRoute = {
  id: string;
  fromDeviceId: string;
  toDeviceId: string;
  cableIds: string[];
  pathSegmentIds: string[];
};
```

## 9. 全链路验收标准

项目达到可用状态时，必须完成以下完整流程：

1. 新建工程。
2. 导入 PNG 底图。
3. 完成两点 CAD 坐标校准。
4. 在图纸上绘制通道拓扑。
5. 设置通道类型。
6. 标记设备节点并录入高度。
7. 创建线缆或线缆组合。
8. 生成线缆路由。
9. 自动推演通道规格。
10. 输入通道敷设深度。
11. BOM 自动生成并随修改实时刷新。
12. 导出 CAD 脚本。
13. 保存工程 JSON。
14. 重新载入工程 JSON。
15. 确认图纸状态、拓扑、设备、路由、BOM 与保存前一致。

## 10. 默认决策

- 默认只支持 `PNG` 底图导入。
- 默认不解析 `.dwg` 或 `.dxf` 文件。
- 默认不建设业务后端。
- 默认以 `.bas` 作为第一阶段 CAD 脚本交付格式。
- 默认采用非线性工作流，允许用户随时返回前面步骤修改。
- 默认优先保证业务链路正确，再优化视觉表现和交互细节。
- 默认所有核心算法和状态计算必须可单元测试。
